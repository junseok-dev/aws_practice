# AWS 서버 및 서비스 관리 가이드

이 가이드는 AWS EC2 인스턴스 자체를 관리하는 방법과, 서버 내부에서 도커(Docker) 기반 서비스를 관리하는 방법을 설명합니다.

---

## 1. 인스턴스 관리 (AWS 콘솔 이용)

서버(PC) 자체를 켜고 끄는 방법입니다. 서버를 끄면 과금이 중단(스토리지 제외)되지만, **공인 IP가 변경될 수 있음**에 유의하세요.

### 서버 켜기 (Start)
1.  [AWS EC2 콘솔](https://ap-northeast-2.console.aws.amazon.com/ec2/home?region=ap-northeast-2#Instances:)에 접속합니다.
2.  사용 중인 인스턴스(예: `aws_practice`)를 선택합니다.
3.  상단의 **[인스턴스 상태]** 버튼을 누릅니다.
4.  **[인스턴스 시작]**을 클릭합니다.
5.  상태가 `실행 중`으로 바뀔 때까지 기다립니다.

### 서버 끄기 (Stop)
1.  인스턴스를 선택합니다.
2.  상단의 **[인스턴스 상태]** 버튼을 누릅니다.
3.  **[인스턴스 중지]**를 클릭합니다. (종료 'Terminate'는 삭제이므로 절대 누르지 마세요!)
    > [!WARNING]
    > 인스턴스를 중지했다가 다시 시작하면 'Public IPv4 주소'가 바뀔 수 있습니다. 이 경우 GitHub Secrets의 `EC2_HOST` 값을 새 주소로 업데이트해야 배포가 정상 작동합니다.

---

## 2. 서비스 관리 (서버 내부 소스 관리)

서버는 켜져 있는데, 돌고 있는 웹 서비스만 재시작하거나 끄고 싶을 때 사용합니다. (SSH 접속 후 수행)

### 서비스 끄기 (Stop Containers)
서버에 접속한 후 아래 명령어를 입력합니다.
```bash
cd ~/aws_practice/backend
sudo docker-compose down
```

### 서비스 켜기 (Start Containers)
코드를 수정하지 않고 단순히 켤 때 사용합니다.
```bash
cd ~/aws_practice/backend
sudo docker-compose up -d
```

### 로그 확인하기 (Monitor Logs)
서버가 잘 작동하는지 실시간으로 확인하고 싶을 때 유용합니다.
```bash
cd ~/aws_practice/backend
sudo docker-compose logs -f
```

## 3. 요금 최적화 가이드 (중요)

AWS 비용을 최소화하고 의도치 않은 과금을 방지하기 위한 핵심 전략입니다.

### 💡 과금이 멈추는 기준
*   **컴퓨팅 비용**: AWS 콘솔에서 인스턴스 상태가 **'중지(Stopped)'**인 시간 동안은 서버 사용료가 부과되지 않습니다.
*   **주의**: 터미널에서 `sudo docker-compose down`으로 서비스만 끄는 것은 **과금이 계속됩니다.** (하드웨어를 계속 점유하고 있기 때문)

### 💡 인스턴스 중지 시 발생하는 일
*   **고정 IP 미사용 시**: 인스턴스를 중지했다가 다시 시작하면 **Public IP 주소가 변경**됩니다.
    *   **조치**: 새 주소를 확인한 후 GitHub Secrets의 `EC2_HOST`와 브라우저 접속 주소를 업데이트해야 합니다.
*   **EBS(디스크) 비용**: 서버를 꺼도 소스 코드를 보존하기 위해 하드디스크 비용(8GB 기준 약 월 1,000원 이하)은 계속 발생합니다.

### 💡 최적의 운영 패턴
1.  **개발 시**: 인스턴스를 켜두고 터미널에서 자유롭게 서비스를 재시작하며 작업합니다.
2.  **작업 종료 시 (하루 이상)**: 반드시 AWS 콘솔에서 인스턴스를 **[중지]** 하세요.
3.  **완전 삭제**: 프로젝트가 완전히 끝나서 더 이상 필요 없다면 **[종료(Terminate)]**를 하세요. (모든 데이터 파괴)
