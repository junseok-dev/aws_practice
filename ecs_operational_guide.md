# AWS ECS(Fargate) 서비스 및 비용 관리 가이드

ECS Fargate는 EC2와 달리 '서버'라는 개념이 없고 '작업(Task)'이라는 컨테이너 단위로 움직입니다. 이 가이드는 서비스를 끄고 켜는 방법과 비용을 최소화하는 방법을 설명합니다.

---

## 1. 서비스 켜고 끄기 (Start/Stop)

ECS Fargate는 인스턴스를 중지하는 대신, 실행하고자 하는 **'원하는 작업 개수(Desired tasks)'**를 조절하여 서비스를 관리합니다.

### 💡 서비스 끄기 (비용 절감 모드)
서버를 물리적으로 삭제하는 것이 아니라, 실행 중인 컨테이너 개수를 0으로 만들어 과금을 방지합니다.
1.  **ECS 콘솔** 접속 -> 생성한 **클러스터** 클릭
2.  **[서비스]** 탭에서 실행 중인 서비스 선택 후 **[수정(Update)]** 클릭
3.  **배포 구성 -> 원하는 작업(Desired tasks)**의 숫자를 **`0`**으로 입력
4.  하단의 **[업데이트]** 버튼 클릭
    - *결과: 실행 중인 컨테이너가 하나씩 꺼지며, 더 이상 컴퓨팅 비용이 발생하지 않습니다.*

### 💡 서비스 켜기 (운영 모드)
1.  위와 동일하게 서비스 **[수정]**으로 진입
2.  **원하는 작업(Desired tasks)**의 숫자를 **`1`** (또는 그 이상)로 입력 후 업데이트
    - *결과: AWS가 자동으로 새 컨테이너를 할당하여 서비스를 재개합니다.*

---

## 2. 요금 최적화 및 과금 방지 전략 (핵심)

Fargate 환경은 여러 서비스가 얽혀있어, 컨테이너만 끈다고 과금이 완전히 멈추지 않습니다. 아래 항목들을 꼭 체크하세요.

### ① Fargate (컴퓨팅) - **작업 개수 0일 때 0원**
*   가장 큰 비용을 차지합니다. 위 방법대로 '원하는 작업 수'를 0으로 만들면 즉시 과금이 중단됩니다.

### ② RDS (데이터베이스) - **주의 필요**
*   **컴퓨팅 중지**: RDS도 인스턴스를 선택하고 **[일시 중지]**할 수 있습니다. (최대 7일간 유지 가능)
*   **주의**: 중지 상태여도 데이터 저장 비용(EBS)은 아주 소액 발생합니다.

### ③ ALB (로드 밸런서) - **고정 비용 발생**
*   로드 밸런서는 컨테이너가 떠 있든 없든 **시간당 고정 비용**이 발생합니다 (약 월 $20 내외).
*   **팁**: 프로젝트를 며칠~몇 주 이상 장기간 사용하지 않는다면 로드 밸런서를 **삭제**하는 것이 가장 확실한 절약 방법입니다. (나중에 다시 만들 수 있습니다.)

### ④ NAT Gateway (사용 금지 권장)
*   가장 무서운 과금 요인입니다. VPC 생성 시 '프라이빗 서브넷'을 만들면 자동으로 생성되는 경우가 많은데, 시간당 비용이 매우 비쌉니다.
*   **해결책**: 현재처럼 **'퍼블릭 서브넷'**에만 자격 증명과 리소스를 배치하여 NAT Gateway 없이 운영하면 네트워크 비용을 0원에 가깝게 유지할 수 있습니다. (현재 저희의 전략입니다.)

---

## 3. 요약: 작업 종료 시 체크리스트

1.  [ ] **ECS 서비스**: Desired tasks를 `0`으로 수정
2.  [ ] **RDS 인스턴스**: [작업] -> [일시 중지] 클릭
3.  [ ] **ALB (선택)**: 단기간 쉴 때는 유지, 장기간(한 달 이상) 쉴 때는 삭제
